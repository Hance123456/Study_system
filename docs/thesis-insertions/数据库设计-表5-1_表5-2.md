# 数据库设计配套表格（可直接粘贴论文）

## 表5-1 核心数据表清单

| 序号 | 数据表名 | 主键 | 核心字段（节选） | 主要用途 |
|---|---|---|---|---|
| 1 | users | id | openid, nickname, created_at | 存储学习用户身份信息，作为学习行为数据的关联主体 |
| 2 | courses | id | name, sort_order, status | 存储课程信息并控制课程展示顺序 |
| 3 | cards | id | course_id, title, content, audio_url, sort_order, status | 存储知识卡片内容与音频资源地址 |
| 4 | quizzes | id | card_id, question_type, question, options, answer, sort_order | 存储卡片关联测验题，支持题型管理与排序 |
| 5 | user_progress | id | user_id, card_id, mastery_level, last_learn_at, next_review_at | 记录用户对卡片的学习掌握与下一次复习时间 |
| 6 | review_plans | id | user_id, card_id, plan_date, stage, is_completed, completed_at | 存储复习任务计划与执行状态（今日/未来） |
| 7 | checkin_records | id | user_id, stat_date, study_duration, cards_learned, cards_reviewed, quiz_count, correct_count | 按日聚合学习统计，用于日历与报表展示 |

---

## 表5-2 关键字段说明

| 字段名 | 所属表 | 类型（示例） | 业务含义 | 备注 |
|---|---|---|---|---|
| id | 全部核心表 | BIGINT / INT | 主键标识 | 自增主键 |
| openid | users | VARCHAR(64) | 微信用户唯一标识 | 建议唯一约束 |
| sort_order | courses/cards/quizzes | INT | 主排序字段 | 与 `id` 组成稳定排序 |
| status | courses/cards/quizzes | TINYINT | 数据状态（启用/禁用） | 列表查询常用过滤字段 |
| course_id | cards | BIGINT | 卡片所属课程 | 外键关联 `courses.id` |
| card_id | quizzes/user_progress/review_plans | BIGINT | 关联卡片 | 外键关联 `cards.id` |
| user_id | user_progress/review_plans/checkin_records | BIGINT | 关联学习用户 | 外键关联 `users.id` |
| audio_url | cards | VARCHAR(500) | 卡片音频访问地址 | 支持上传音频与 TTS 结果 |
| question_type | quizzes | TINYINT / INT | 题型标识 | 用于控制前端渲染与判定 |
| options | quizzes | JSON / TEXT | 客观题选项集合 | 题型相关字段 |
| answer | quizzes | TEXT / VARCHAR | 标准答案 | 支持单选/多选/判断等 |
| mastery_level | user_progress | INT | 掌握度/熟练度 | 参与复习策略计算 |
| plan_date | review_plans | DATE | 计划复习日期（按天） | 今日/即将到来判断依据 |
| next_review_at | user_progress | DATETIME | 下一次复习时间（具体时刻） | 与 `plan_date` 语义不同 |
| is_completed | review_plans | TINYINT | 复习任务完成标记 | 0 未完成，1 已完成 |
| completed_at | review_plans | DATETIME | 实际完成时间 | 用于审计与统计 |
| stat_date | checkin_records | DATE | 统计日期 | 日历主键字段之一 |
| study_duration | checkin_records | INT | 当日学习时长 | 建议单位统一（秒/分钟） |
| cards_learned | checkin_records | INT | 当日学习卡片数 | 统计指标 |
| cards_reviewed | checkin_records | INT | 当日复习卡片数 | 统计指标 |
| quiz_count | checkin_records | INT | 当日测验次数 | 统计指标 |
| correct_count | checkin_records | INT | 当日答对数量 | 统计指标 |

## 表5-3 关键索引与约束设计

| 序号 | 表名 | 类型 | 字段/约束 | 设计目的 |
|---|---|---|---|---|
| 1 | users | 唯一约束 | `UNIQUE(openid)` | 保证同一微信用户不重复入库 |
| 2 | cards | 外键约束 | `course_id -> courses.id` | 保证卡片必须属于有效课程 |
| 3 | quizzes | 外键约束 | `card_id -> cards.id` | 保证题目与卡片关联有效 |
| 4 | user_progress | 外键约束 | `user_id -> users.id` | 保证进度数据归属于有效用户 |
| 5 | user_progress | 外键约束 | `card_id -> cards.id` | 保证进度数据指向有效卡片 |
| 6 | review_plans | 外键约束 | `user_id -> users.id` | 保证复习计划归属于有效用户 |
| 7 | review_plans | 外键约束 | `card_id -> cards.id` | 保证复习计划指向有效卡片 |
| 8 | checkin_records | 外键约束 | `user_id -> users.id` | 保证打卡记录归属于有效用户 |
| 9 | review_plans | 普通索引 | `(user_id, plan_date, is_completed)` | 加速“今日待复习/即将到来”查询 |
| 10 | user_progress | 普通索引 | `(user_id, card_id)` | 加速用户-卡片进度查询与更新 |
| 11 | cards | 普通索引 | `(course_id, sort_order, id)` | 加速课程卡片列表稳定排序 |
| 12 | quizzes | 普通索引 | `(card_id, sort_order, id)` | 加速卡片题目列表稳定排序 |
| 13 | checkin_records | 普通索引 | `(user_id, stat_date)` | 加速按日期统计与日历详情查询 |

**说明：**

- 若数据库已开启外键级联（`ON DELETE CASCADE`），删除课程/卡片时可自动清理关联记录；若未开启，则需在业务层执行显式清理逻辑。  
- 复习相关查询是系统高频路径，`review_plans` 与 `user_progress` 的组合索引应优先保证。  
- 排序相关索引需与实际 SQL `ORDER BY` 字段顺序保持一致，以获得更稳定的查询性能表现。
